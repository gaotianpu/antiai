# Parking Lot — 搁置清单管理

## 触发信号

| 用户用语 | 语义 | 操作 |
|:---|:---|:---|
| "搁置" / "先不用了" | 推迟执行，非放弃 | 追加到 backlog |
| "先保存下来" / "先记着" | 备忘但不执行 | 追加到 backlog（标记 📌） |
| 对话中提及某个搁置过的事项 | 可能已到执行时机 | 查 backlog 恢复上下文 |

---

## 操作流程

### 1. 搁置追加（用户说"搁置"时）

```bash
# 1. 读取当前 backlog.md
cat wiki/backlog.md

# 2. 确定分类归属
# 扫描 backlog.md，根据事项性质放入对应分类下：
#   - 有明确搁置理由的 → 放入主表格（标记 ⏸️ ⏳ 🧩 📌）
#   - 是已创建文档的关联事项 → 放入"已创建文档"之前或之后
#   - 全新类别 → 新增分类标题

# 3. 追加行（仅在表格末尾追加，不重新生成全文）
# 格式：
# | 事项描述 | YYYY-MM-DD | ⏸️ 暂缓 | 搁置原因 / 恢复条件 |
```

**搁置理由标记**：

| 标记 | 含义 | 适用场景 |
|:---|:---|:---|
| ⏸️ 暂缓 | 暂时不做 | 优先级低、资源不足 |
| ⏳ 等待前置 | 依赖某前置条件 | 需等另一事项完成、依赖数据/工具就绪 |
| 🧩 碎片化 | 概念不成熟/太零散 | 单一 paper 提及但无展开价值 |
| 📌 备忘 | 仅记录不执行 | 观察项、未来参考 |

### 2. 会话启动扫描

**当用户开启新会话时（session_start），检查 backlog.md 中是否有事项可以解冻**：

```bash
# 读取 backlog
cat wiki/backlog.md

# 检查条件：
# - ⏳ 等待前置：前置条件是否已满足？
# - ⏸️ 暂缓：当前会话任务是否与搁置事项相关？
# - 📌 备忘：是否有新进展触发此备忘？

# 如有可解冻项，在回复末尾提示用户
```

**解冻判断示例**：

```
📋 backlog 中有一项待关注：
  · ⏳ 2026-05-16 「A 级公式质量审查人工确认」—— 等待 PDF 确认
  当前已读过 raw/ 下的 PDF，可推进此审查，需要处理吗？
```

### 3. 搁置项上下文恢复

**当提及某个已搁置事项时**：

```bash
# 1. 从 backlog.md 中找到该事项行
grep -n "关键词" wiki/backlog.md

# 2. 查看当时对话日志（如果有 log 页面记录）
ls wiki/changelog/ | tail -5
```

- 如果用户问"之前说的那个 XX 的事" → 先去 backlog.md 找，不要凭空回忆
- 找到后输出：搁置日期 + 原因 + 当时的上下文摘要

### 4. 搁置项完成/删除

当用户明确表示一个搁置事项已完成或不再需要时：

```bash
# 从 backlog.md 中删除该行（或整行删除）
# 注意保持表格对齐，不要留空行
```

## 补充规范

- **只追加不重写**：backlog.md 的主表格区只追加行，不重新生成。只有新增分类时可以用 write 全量重写
- **日期格式**：`YYYY-MM-DD`
- **表格对齐**：追加时保持 `| :--- | :---: | :---: | :--- |` 格式对齐
- **与 changelog 联动**：搁置操作本身也记录在 `wiki/changelog/` 的操作日志中
