## 用户习惯备忘录

| 习惯 | 表现 | Agent 应如何响应 |
|:---|:---|:---|
| **信息密度优先** | 索引不只是文件名，要求提取核心摘要和关键词 | 首次创建索引或大规模更新时，主动从 wiki source 补充摘要和标签 |
| **先看再定** | 喜欢先看方案/结果再做决策（如先问含义再决定建页；先看分类再决定按年份） | 不确定方向时先出方案预览，等用户确认后再执行 |
| **擅长折中** | 被问到"全部还是部分"时倾向选中间方案 | 给出明确选项时附上折中方案 |
| **按年份感知** | 从主题分类改为按年份正序分组 | 涉及论文时间线时默认按年份正序组织 |
| **规则要沉淀到正确位置** | 要求将维护规则追加到 skill 而非 schema | 涉及工作流流程的规则写到对应 skill；操作经验写到 `schema/best_practices.md` |
| **结果要落地** | 讨论产出的清单要求保存为文件 | 超过 10 项的调研/规划结果自动创建 synthesis 页固化 |
| **只 commit 不 push** | 每次要求提交但不推送 | 严格遵守"逐单元 commit，会话结束再决定推送" |
