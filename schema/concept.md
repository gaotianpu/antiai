# 概念识别与维护规范

## 概念定义
**概念**: 抽象的思想、理论、原则或模式，用于组织和理解特定领域的知识。概念比实体更抽象，通常代表一类思想或模式。

## 概念与实体的区别

| 特征 | 概念 | 实体 |
|------|------|------|
| **抽象程度** | 高，代表思想或模式 | 低，代表具体存在 |
| **可识别性** | 通过定义识别 | 通过名称唯一标识 |
| **示例** | 卷积、注意力机制 | ResNet、Google DeepMind |
| **文件位置** | `wiki/concepts/` | `wiki/entities/` |

## 概念分类体系

### 1. 理论概念
- 基础理论、原理、定律
- 示例: `attention_mechanism.md`, `information_bottleneck.md`

### 2. 方法概念
- 分析方法、研究框架、方法论
- 示例: `backpropagation.md`, `gradient_descent.md`

### 3. 模式概念
- 重复出现的模式、趋势、规律
- 示例: `emergent_abilities.md`, `scaling_law.md`

### 4. 原则概念
- 指导原则、最佳实践、规范
- 示例: `responsible_ai.md`, `data_governance.md`

### 5. 关系概念
- 实体间的关系、相互作用
- 示例: `bias_variance_tradeoff.md`, `capacity_vs_generalization.md`

## 文件命名规范
- **格式**: 蛇形命名法 (snake_case)
- **规则**: 使用描述性名称，反映概念本质
- **示例**: `transformer_architecture.md`, `reinforcement_learning.md`

## 工作流程

### 1. 概念识别流程
1. **资料分析** - 从原始资料中提取抽象思想
2. **概念提取** - 识别重复出现的模式、原则
3. **抽象层次判断** - 确定概念的抽象程度
4. **关系建立** - 建立概念间的层次和关联关系

### 2. 概念创建/更新流程
1. **唯一性检查** - 检查是否已存在相同或相似概念（调用 `schema/concept_dedup.md`）
2. **抽象程度评估** - 确定概念的抽象层次
3. **关系网络构建** - 建立与相关概念、实体的关系
4. **页面创建/更新** - 创建新页面或更新现有页面
5. **索引更新** - 更新 `wiki/concepts/index.md`

### 3. 概念演化处理
1. **版本演进** - 记录概念定义的变化
2. **范围扩展** - 处理概念应用范围的扩大
3. **内涵深化** - 处理概念理解的深化
4. **关系调整** - 调整概念间的关系网络

## 质量保证

### 概念质量标准
- **清晰性**: 定义明确，边界清晰
- **一致性**: 内部逻辑一致，不自相矛盾
- **完整性**: 涵盖概念的所有重要方面
- **实用性**: 对理解领域知识有实际帮助
- **演进性**: 能够记录和反映概念的演化

### 关系网络质量
- **层次合理**: 抽象层次划分合理
- **关联准确**: 概念间关系准确反映知识结构
- **网络连通**: 概念网络具有良好的连通性
- **边界清晰**: 不同概念间边界清晰

## 维护策略

### 定期维护
1. **概念审查** - 每月审查概念的定义和关系
2. **网络优化** - 优化概念关系网络
3. **抽象层次调整** - 根据理解深化调整抽象层次
4. **实例更新** - 更新概念的实例和案例

### 演化跟踪
1. **版本记录** - 记录概念定义的重大变化
2. **影响分析** - 分析概念变化对相关概念的影响
3. **关系更新** - 更新因概念演化而变化的关系

## 与实体的协作

### 概念-实体映射
1. **实例化关系** - 概念应有具体的实体实例
2. **抽象化关系** - 实体应归属于相应的概念
3. **双向链接** - 概念和实体间建立双向链接

### 协同更新
1. **实体更新触发概念更新** - 新实体可能影响相关概念
2. **概念更新触发实体重组** - 概念演化可能导致实体重新分类
3. **一致性检查** - 定期检查概念和实体的一致性
